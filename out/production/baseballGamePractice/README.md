# 🎯 숫자야구 게임 – 요구사항 명세서


## 📌 1. 기능 개요


컴퓨터는 **1~9 사이 서로 다른 3개의 숫자**를 임의로 생성한다.
사용자는 3자리 숫자를 입력하며, 컴퓨터 숫자와 비교하여 다음 규칙으로 결과를 안내한다.


* **스트라이크 (S)**: 숫자와 위치가 모두 같음
* **볼 (B)**: 숫자는 맞지만 위치는 다름
* **낫싱 (Nothing)**: 하나도 맞는 숫자가 없음
* 정확히 맞추면 **3스트라이크 → 게임 클리어**


---


# 📌 2. 요구사항 상세


## ✔ 2.1 컴퓨터 숫자 생성


* 서로 다른 3개의 숫자를 무작위 생성
* 숫자 범위: 1~9
* 중복 불가


## ✔ 2.2 사용자 입력


사용자는 **3자리 숫자**를 입력한다.
(예: `125`, `987`)


### ⛔ 입력 예외 조건


아래 상황은 모두 **예외**이며 재입력 요청 필요:


* 숫자가 아님 (문자, 공백 포함)
* 3자리 미입력(1자리, 2자리, 4자리 이상)
* 0 포함 또는 범위 벗어남 (1~9만 가능)
* 중복 숫자 입력


예) `012`, `a23`, `112`, `9 2 3` → 예외


---


# 📌 3. 판정 규칙


입력값과 컴퓨터 숫자를 비교하여 스트라이크/볼 계산.


예)


* 컴퓨터: 425


* 사용자: 123
  → 1 스트라이크 (2 위치 일치)


* 컴퓨터: 425


* 사용자: 546
  → 1 볼 (5은 있지만 위치 다름)


* 컴퓨터: 425


* 사용자: 789
  → 낫싱

# 📌 4. 출력 요구사항


## ✔ 실행 과정 출력 예시 (문구 고정)


```
숫자 야구 게임을 시작합니다.
숫자를 입력해주세요 : 123
1볼 1스트라이크

3스트라이크
3개의 숫자를 모두 맞히셨습니다! 게임 종료
게임을 새로 시작하려면 1, 종료하려면 2를 입력하세요.
```


## ✔ 출력 규칙


* 볼이 0 → `x스트라이크`만 출력
* 스트라이크가 0 → `x볼`만 출력
* 둘 다 0 → `낫싱` 출력


예)


* `1볼`
* `2스트라이크`
* `낫싱`
* `1볼 2스트라이크`

# 📌 6. 아키텍처 요구사항 (MVC 맞춤)

* Game: 판정 로직, 결과(볼/스트라이크 계산)

* InputView: 입력 요청 + 입력값 받기
